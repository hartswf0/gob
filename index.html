Interpretation Of The Overall System

The design separates discovery, excavation, inference, and interface. The separation is not cosmetic; it prevents premature closure and keeps evidence density high before decision framing.

01 - Divergence Engine

Expands the search space. It forbids analysis so coverage stays wide and non-committal.

02 - Evidence Construction

Builds thick dossiers with contradictions intact, turning one site into an evidentiary object.

03 - Structural Interpretation

Extracts incentives, tensions, and implications from the dossier, forcing a decisive "so what."

04 - Research Instrument Layer

Transforms site inventories into usable interfaces so exploration scales beyond raw text.

Handoff Contract (Fortification)

Each step has an input contract and an output contract. Keep these boundaries intact so later steps remain reliable.

Step Required Input Required Output Feeds Into
01 Domain/question/territory Unranked site list with coordinates + relevance 02 (choose one site) and 04 (use full list)
02 Single site from 01 Five-section dossier with contradictions preserved 03
03 Dossier (or equivalent document) Insights, tensions, core implication, missing inquiry Decision and next investigation cycle
04 Site list (usually from 01) Standalone exploration interface Ongoing browsing, filtering, and site selection

Operational Guardrails

  • Do not skip directly from 01 to 03; 03 expects a dense dossier, not a site list.
  • Do not summarize during 02; contradictions and unresolved threads are required data.
  • Freeze outputs between steps; avoid quietly rewriting artifacts before handoff.
  • Log source scope, date, and prompt version for each run so results remain auditable.
  • Treat 04 as interface infrastructure, not analytical evidence.

Recommended Sequence

  1. Run 01-OS.md on a topic to map the terrain.
  2. Select one site and run 02-OS.md to excavate depth.
  3. Run 03-OS.md on the dossier to derive implications.
  4. Use 04-OS.md when you want interactive browsing of site inventories.

System summary: this is a methodological pipeline, not a single prompt. Reliability comes from strict stage boundaries and explicit handoffs.
